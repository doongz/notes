# 大模型量化技术：PTQ、QAT、AWQ 全解析

大模型量化是核心的模型压缩技术，通过将模型中高精度浮点数值（FP32/FP16/BF16）映射到低比特整数（INT8/INT4/INT2），在最小化精度损失的前提下，**大幅降低显存占用、提升推理速度、降低部署门槛**，是大模型从实验室走向工业落地、消费级硬件部署的核心技术。

主流量化技术分为两大基础范式：**训练后量化（PTQ）** 和**量化感知训练（QAT）**；而 AWQ 是针对大模型低比特场景优化的、当前最主流的 PTQ 进阶方案。

## 一、PTQ（Post-Training Quantization，训练后量化）

PTQ 是工业界最基础、最成熟的量化方案，核心是在模型预训练完成后，无需修改模型结构、无需重新训练权重，仅通过少量无标签校准数据，完成高精度到低精度的数值映射。

### 核心原理与流程

1. **核心逻辑**：通过校准数据统计各层权重、激活值的数值分布，计算量化的缩放因子（scale）和零点（zero point），完成浮点到整型的线性映射，全程无权重更新。
2. **主流分类**：
   - 动态 PTQ：仅量化权重，激活值在推理时实时计算量化参数，实现极简，显存节省和加速效果有限；
   - 静态 PTQ（工业主流）：提前用校准数据集（100-1000 条与任务分布一致的样本）完成前向传播，统计数值分布并优化量化参数，推理时直接使用固定参数，加速效果更优。
3. **标准流程**：加载预训练浮点模型 → 输入校准数据统计数值分布 → 计算最优量化参数 → 执行权重 / 激活量化 → 导出量化模型部署。

### 核心优缺点与适用场景

✅ **优点**：零训练成本，算力开销极低，普通 CPU 即可完成；工程化成熟，所有主流推理框架原生支持；INT8 场景下精度损失可忽略，落地门槛极低。

❌ **缺点**：INT4 及以下低比特量化时，对权重 / 激活的离群值极度敏感，精度下降显著，甚至出现输出崩塌；对代码生成、数学推理等复杂任务适配性差。

📌 **适用场景**：工业界大模型服务端 INT8 大规模部署；快速原型验证、算力资源紧张的项目；对精度要求不极致的通用文本场景。

## 二、QAT（Quantization-Aware Training，量化感知训练）

QAT 是量化精度的天花板方案，核心是在模型微调阶段，将量化误差完整模拟进训练流程，让模型在训练时就主动适配低比特量化的噪声，学习对量化不敏感的权重分布，最终在极低比特下仍保持接近全精度的性能。

### 核心原理与流程

1. **核心机制**：
   - 伪量化操作：前向传播时，对权重和激活值执行 “量化→反量化” 的伪量化，模拟真实部署的舍入误差，将量化噪声纳入损失函数；
   - 梯度适配：反向传播时，通过**直通估计器（STE）** 绕过量化操作的不可导问题，正常更新权重，让模型主动修正量化带来的精度损失。
2. **大模型主流优化方案**：普遍采用**LoRA-QAT**，冻结主干权重，仅对 LoRA 低秩适配器执行量化感知训练，在极低训练成本下实现接近全参数 QAT 的精度；工业界常搭配知识蒸馏（KD-QAT），进一步降低精度损失。
3. **标准流程**：加载预训练浮点模型 → 插入伪量化节点搭建量化训练图 → 用任务数据集执行低学习率微调 → 训练完成后融合量化节点 → 导出全量化模型部署。

### 核心优缺点与适用场景

✅ **优点**：精度上限极高，INT4/INT2 超低比特量化下，精度损失可控制在 1% 以内，远超 PTQ 类方案；能有效解决离群值、复杂任务的量化精度崩塌问题。

❌ **缺点**：训练 / 算力成本极高，需要完整微调流程、大量标注数据和高性能 GPU 算力；工程化门槛高，不同模型、硬件的算子适配复杂，调参难度大。

📌 **适用场景**：对精度、延迟、可靠性要求极致的核心业务场景；端侧 / 边缘设备的超低比特部署；自动驾驶、医疗 AI 等高风险领域的模型量化。

## 三、AWQ（Activation-aware Weight Quantization，激活感知权重量化）

AWQ 是 2023 年提出的、专为大语言模型优化的 INT4 低比特 PTQ 进阶方案，核心突破是基于激活值分布识别关键权重通道，通过无损缩放保护核心权重，在零训练成本的前提下，实现远超传统 PTQ 的精度，甚至接近 QAT 的效果，是当前消费级硬件部署大模型的事实标准之一。

### 核心原理与流程

1. **核心洞见**：大模型中仅 0.1%-1% 的权重通道，对应激活值的幅度更大，对模型输出起决定性作用；传统 PTQ 均匀量化所有权重，会导致这些关键通道的量化误差被放大，最终引发精度崩塌。
2. **核心优化**：
   - **激活感知通道缩放**：通过校准数据统计每个权重通道对应的激活值幅度，识别关键通道；量化前对关键权重通道乘以缩放因子放大，同时对对应输入激活除以相同因子，数学上不改变模型输出，但大幅降低了关键权重的量化舍入误差，同时避免了混合精度带来的硬件不友好问题；
   - **组级量化优化**：采用 group-wise 量化（通常每组 128 个权重共享一个缩放因子），相比传统逐通道量化粒度更细、精度更高，同时兼容主流 GPU 的 INT4 指令集，推理效率拉满；
   - **轻量级误差优化**：通过网格搜索最优缩放因子，最小化量化前后的输出误差，全程无需反向传播和训练。
3. **标准流程**：加载预训练 FP16 模型 → 输入校准数据统计激活值分布，识别关键权重通道 → 执行激活感知通道缩放 → 完成 INT4 组级量化 → 导出量化模型部署。

### 核心优缺点与适用场景

✅ **优点**：零训练成本，和基础 PTQ 一致，仅需少量校准数据；INT4 量化精度远超传统 PTQ，多数任务下接近 QAT 的效果；工程化成熟，主流框架原生支持，硬件友好，推理速度优异；显存压缩极致，7B 模型 4bit 量化后仅需 3-4GB 显存，消费级显卡即可流畅运行。

❌ **缺点**：主打 INT4 场景，INT8 场景相比传统 PTQ 无明显优势；极端低比特（INT2）场景下，精度仍与 QAT 有差距。

📌 **适用场景**：消费级显卡 / 个人设备部署大模型的首选方案；中小团队大模型服务端批量部署，追求显存节省与推理效率，同时不想付出训练成本的场景；开源大模型的轻量化部署。

## 四、核心技术对比与选型建议

| 技术维度         | PTQ（基础训练后量化）          | QAT（量化感知训练）                     | AWQ（激活感知权重量化）            |
| ---------------- | ------------------------------ | --------------------------------------- | ---------------------------------- |
| **核心范式**     | 训练后量化                     | 训练中适配量化                          | 训练后量化（PTQ 进阶）             |
| **训练成本**     | 零训练，仅需校准（分钟级）     | 高，需完整微调（小时 / 天级）           | 零训练，仅需校准（分钟级）         |
| **数据需求**     | 数百条无标签校准数据           | 完整标注训练数据集                      | 数百条无标签校准数据               |
| **精度表现**     | INT8 优秀，INT4 及以下精度崩塌 | 全比特位精度天花板，INT2 仍可保持高性能 | INT4 精度远超基础 PTQ，接近 QAT    |
| **工程门槛**     | 极低，开箱即用                 | 高，需适配训练流程与硬件算子            | 低，主流框架原生支持               |
| **核心优势**     | 成熟稳定，落地成本最低         | 精度极致，适配超低比特与高可靠场景      | 零成本下实现 INT4 高精度，硬件友好 |
| **核心短板**     | 低比特精度差                   | 训练与落地成本极高                      | 仅 INT4 场景优势显著               |
| **主流适用位宽** | INT8                           | INT4/INT2                               | INT4                               |

### 最终选型建议

1. 工业界服务端大规模部署：INT8 场景优先选择**静态 PTQ**，成熟稳定，落地成本最低；
2. 消费级硬件 / 个人部署、中小团队服务端部署：INT4 场景优先选择**AWQ**，是当前精度、速度、成本的最优解；
3. 端侧 / 边缘设备超低比特部署、高可靠核心业务场景：有充足算力与数据的前提下，选择**LoRA-QAT**，实现精度与压缩率的极致平衡。
