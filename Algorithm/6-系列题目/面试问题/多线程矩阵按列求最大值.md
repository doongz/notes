# 多线程矩阵按列求最大值

这是一道典型的**多线程矩阵按列求最大值**的并行计算面试题，我会先讲思路，再给出完整可运行的实现。

---

## 一、问题分析与思路讲解
### 1. 问题拆解
- 输入：一个 `M × N` 的二维矩阵（在内存中按行存储为一维数组 `input[M*N]`）
- 输出：长度为 `N` 的数组 `output[N]`，其中 `output[j]` 是第 `j` 列的最大值
- 约束：使用 `n` 个线程并行计算，每个线程由 `tid`（线程号）标识

### 2. 并行划分策略（核心）
按**列划分任务**是最自然的方式：
- 总共有 `N` 列，把这些列平均分配给 `n` 个线程处理
- 每个线程 `tid` 负责处理列号 `j` 满足 `j % n == tid` 的所有列
- 这样做的好处：
  - 无数据竞争：每个线程只写自己负责的 `output[j]`，不需要加锁
  - 缓存友好：按列访问时，虽然跨行使局部性差，但任务划分逻辑简单，且多线程下无同步开销

### 3. 内存访问优化（针对题目中的两个场景）
| 场景 | M（行数） | N（列数） | 线程数n | 优化重点 |
|------|-----------|-----------|---------|----------|
| 一 | 1,000,000 | 2048 | 32 | N大，列数多，线程负载均衡优先 |
| 二 | 10,000,000 | 64 | 1024 | M大，行数多，减少跨行使的缓存失效 |

针对大M的场景，我们可以在代码中加入**循环展开**或**分块预取**，提升内存访问效率。

## 二、C++ 完整实现（含多线程任务划分）
```cpp
#include <iostream>
#include <vector>
#include <thread>
#include <algorithm>
#include <cassert>

/**
 * @brief 多线程求矩阵每列最大值（每个线程处理 tid 对应的列）
 * 
 * @param input 输入矩阵，M行N列，行优先存储
 * @param output 输出数组，长度N，每列的最大值
 * @param M 矩阵行数
 * @param N 矩阵列数
 * @param n 总线程数
 * @param tid 当前线程号（0~n-1）
 */
void func(const float* input, float* output, int M, int N, int n, int tid) {
    // 1. 边界检查（面试加分项）
    assert(input != nullptr && output != nullptr);
    assert(M > 0 && N > 0 && n > 0 && tid >= 0 && tid < n);

    // 2. 线程负责的列：按列划分，tid负责 j % n == tid 的列
    for (int j = tid; j < N; j += n) {
        // 3. 初始化当前列的最大值为负无穷
        float col_max = -1e10f;

        // 4. 遍历当前列的所有元素（第j列的元素是 input[i*N + j]，i从0到M-1）
        for (int i = 0; i < M; ++i) {
            float val = input[i * N + j];
            if (val > col_max) {
                col_max = val;
            }
        }

        // 5. 写入结果（无数据竞争，每个线程只写自己负责的output[j]）
        output[j] = col_max;
    }
}

// 测试主函数（模拟题目中的两个场景）
int main() {
    // 场景1：M=1e6, N=2048, 线程数32
    const int M1 = 1000000;
    const int N1 = 2048;
    const int n1 = 32;

    // 场景2：M=1e7, N=64, 线程数1024
    const int M2 = 10000000;
    const int N2 = 64;
    const int n2 = 1024;

    // 为了演示，这里用小矩阵测试逻辑
    const int test_M = 3;
    const int test_N = 4;
    const int test_n = 2;
    float test_input[test_M * test_N] = {
        1.0f, 3.0f, 5.0f, 2.0f,
        4.0f, 2.0f, 1.0f, 7.0f,
        0.0f, 5.0f, 3.0f, 4.0f
    };
    float test_output[test_N] = {0};

    std::vector<std::thread> threads;
    for (int tid = 0; tid < test_n; ++tid) {
        threads.emplace_back(func, test_input, test_output, test_M, test_N, test_n, tid);
    }

    for (auto& t : threads) {
        t.join();
    }

    // 打印结果：预期每列最大值为 [4.0, 5.0, 5.0, 7.0]
    std::cout << "Test case output: ";
    for (int j = 0; j < test_N; ++j) {
        std::cout << test_output[j] << " ";
    }
    std::cout << std::endl;

    return 0;
}
```

# 优化 循环交换，变列遍历为行遍历

面试官核心考察CPU 缓存行利用率，优化核心是「将跳读改为顺序读，最大化 cache 命中率」，分层次优化： 基础访存优化（必答）：循环交换，变列遍历为行遍历 问题根源：原逻辑最内层循环遍历行，相邻两次内存访问间隔 n 个元素，CPU cache line（64 字节）仅用了 4 字节（1 个 float），带宽浪费 93.75%。 优化方案：交换内外层循环，最内层循环遍历同一行的连续列，实现内存顺序访问，cache line 数据 100% 利用。

### 一、核心问题根因复盘（面试官考察的底层逻辑）
原列优先遍历的致命访存问题：
- 访存模式：`for(列j) { for(行i) { input[i*N + j] } }`，相邻两次内存访问步长为 `N*4字节`（1个float占4字节）
- 缓存浪费：CPU Cache Line固定为64字节，可容纳16个float；原模式下每次预取的Cache Line仅用1个float，**有效带宽利用率仅6.25%，93.75%的带宽完全浪费**
- 性能瓶颈：大M场景下（如100万/1000万行），程序完全受限于内存带宽，CPU计算能力完全无法发挥

### 二、分层次优化实现（严格贴合面试官考察要求）
所有实现严格遵循题目给定的函数签名，兼容多线程无锁并行，核心围绕「顺序访存、最大化Cache Line利用率」展开。

#### 🔴 基础优化版（必答，核心解决跳读→顺序读）
**核心优化动作**：
1.  **循环交换**：将「列循环在外、行循环在内」改为「行循环在外、列循环在内」，最内层实现同一行的连续内存访问
2.  **连续列任务划分**：每个线程负责一段连续的列区间，保证最内层循环是步长4字节的纯顺序读，Cache Line 100%利用
3.  **局部最大值缓存**：线程内维护局部max数组，避免频繁写全局output造成的Cache写污染，最后一次性写回结果
4.  **无锁并行**：每个线程仅读写自己负责的列区间，无数据竞争，无需加锁

```c
#include <iostream>
#include <vector>
#include <thread>
#include <algorithm>
#include <cassert>
#include <limits>

/**
 * @brief 精炼版基础优化：循环交换+顺序访存，Cache Line 100%利用
 * 假设 N 能被 n 整除，代码更简洁，面试必答核心版本
 */
void func(const float* input, float* output, int M, int N, int n, int tid) {
    // 1. 边界检查
    assert(input != nullptr && output != nullptr);
    assert(M > 0 && N > 0 && n > 0 && tid >= 0 && tid < n);
    assert(N % n == 0); // 假设 N 能被 n 整除

    // 2. 连续列划分：每个线程负责 [col_start, col_end) 的连续列
    int cols_per_thread = N / n;
    int col_start = tid * cols_per_thread;
    int col_end = col_start + cols_per_thread;

    // 3. 线程局部最大值数组：栈上分配，命中 L1 Cache
    float* local_max = (float*)alloca(cols_per_thread * sizeof(float));
    for (int i = 0; i < cols_per_thread; ++i) {
        local_max[i] = -std::numeric_limits<float>::max();
    }

    // 4. 核心优化：循环交换，行优先遍历，顺序访存
    for (int i = 0; i < M; ++i) {
        const float* row_ptr = input + i * N; // 当前行起始地址
        // 最内层：连续列访问，步长 4 字节，Cache Line 100% 利用
        for (int j = 0; j < cols_per_thread; ++j) {
            float val = row_ptr[col_start + j];
            local_max[j] = std::max(local_max[j], val);
        }
    }

    // 5. 一次性写回结果
    for (int j = 0; j < cols_per_thread; ++j) {
        output[col_start + j] = local_max[j];
    }
}

// 测试主函数
int main() {
    const int test_M = 3;
    const int test_N = 4;
    const int test_n = 2; // 4 能被 2 整除
    float test_input[test_M * test_N] = {
        1.0f, 3.0f, 5.0f, 2.0f,
        4.0f, 2.0f, 1.0f, 7.0f,
        0.0f, 5.0f, 3.0f, 4.0f
    };
    float test_output[test_N] = {0};

    std::vector<std::thread> threads;
    for (int tid = 0; tid < test_n; ++tid) {
        threads.emplace_back(func, test_input, test_output, test_M, test_N, test_n, tid);
    }
    for (auto& t : threads) {
        t.join();
    }

    // 预期输出：4 5 5 7
    std::cout << "列最大值结果：";
    for (int j = 0; j < test_N; ++j) {
        std::cout << test_output[j] << " ";
    }
    std::cout << std::endl;

    return 0;
}
```

解决N不能被n整除的负载均衡问题

```cpp
#include <iostream>
#include <vector>
#include <thread>
#include <algorithm>
#include <cassert>
#include <limits>

/**
 * @brief 基础优化版：循环交换+顺序访存，Cache Line 100%利用
 * 完全匹配面试官要求的核心优化点，面试必答版本
 */
void func(const float* input, float* output, int M, int N, int n, int tid) {
    // 1. 边界合法性检查（工程严谨性加分项）
    assert(input != nullptr && output != nullptr);
    assert(M > 0 && N > 0 && n > 0 && tid >= 0 && tid < n);

    // 2. 连续列区间划分：保证线程负责的列是连续的，为顺序访存铺路
    // 解决N不能被n整除的负载均衡问题
    int cols_per_thread = N / n;
    int remainder = N % n;
    int col_start = tid * cols_per_thread + std::min(tid, remainder);
    int col_end = col_start + cols_per_thread + (tid < remainder ? 1 : 0);
    int thread_col_num = col_end - col_start;

    // 无列可处理的线程直接退出（兼容场景二：线程数>列数的情况）
    if (thread_col_num <= 0) return;

    // 3. 线程局部最大值数组：栈上分配，命中L1 Cache，避免频繁写全局内存
    float* local_max = (float*)alloca(thread_col_num * sizeof(float));
    for (int i = 0; i < thread_col_num; ++i) {
        local_max[i] = -std::numeric_limits<float>::max();
    }

    // 4. 核心优化：循环交换，行优先遍历，顺序访存
    // 外层循环遍历行，内层循环遍历当前线程负责的连续列
    for (int i = 0; i < M; ++i) {
        // 当前行的起始地址，后续访问都是基于该地址的连续偏移
        const float* row_ptr = input + i * N;
        // 最内层循环：连续列访问，步长4字节，完全顺序读，Cache Line 100%利用
        for (int j = 0; j < thread_col_num; ++j) {
            float val = row_ptr[col_start + j];
            local_max[j] = std::max(local_max[j], val);
        }
    }

    // 5. 一次性写回结果：减少全局内存写操作，降低Cache写回开销
    for (int j = 0; j < thread_col_num; ++j) {
        output[col_start + j] = local_max[j];
    }
}

// 测试主函数，兼容题目两个场景
int main() {
    // 场景1：M=1e6, N=2048, 线程数32
    // 场景2：M=1e7, N=64, 线程数1024
    // 小矩阵功能测试
    const int test_M = 3;
    const int test_N = 4;
    const int test_n = 2;
    float test_input[test_M * test_N] = {
        1.0f, 3.0f, 5.0f, 2.0f,
        4.0f, 2.0f, 1.0f, 7.0f,
        0.0f, 5.0f, 3.0f, 4.0f
    };
    float test_output[test_N] = {0};

    std::vector<std::thread> threads;
    for (int tid = 0; tid < test_n; ++tid) {
        threads.emplace_back(func, test_input, test_output, test_M, test_N, test_n, tid);
    }
    for (auto& t : threads) {
        t.join();
    }

    // 预期输出：4 5 5 7
    std::cout << "列最大值结果：";
    for (int j = 0; j < test_N; ++j) {
        std::cout << test_output[j] << " ";
    }
    std::cout << std::endl;

    return 0;
}
```

#### 🟡 进阶缓存优化版（面试加分项，针对超大M场景）
在基础版之上，针对M=1000万行的超大矩阵场景，解决「行数据无法全部放入L3缓存」的问题，进一步提升缓存命中率。
**核心新增优化**：
- **行分块（Tiling）**：将M行切分为固定大小的块，保证当前处理的行块数据能完全放入CPU L2/L3缓存，避免缓存频繁失效
- **编译期常量块大小**：适配CPU缓存大小，编译器可自动做循环展开优化
- **restrict关键字**：告诉编译器指针无别名，开启更强的编译优化

```cpp
// 适配CPU L2缓存的块大小，可根据CPU型号调整（一般256/512/1024行）
#define ROW_BLOCK_SIZE 512

void func_advanced(const float* __restrict__ input, float* __restrict__ output, int M, int N, int n, int tid) {
    assert(input != nullptr && output != nullptr);
    assert(M > 0 && N > 0 && n > 0 && tid >= 0 && tid < n);

    // 连续列划分（同基础版）
    int cols_per_thread = N / n;
    int remainder = N % n;
    int col_start = tid * cols_per_thread + std::min(tid, remainder);
    int col_end = col_start + cols_per_thread + (tid < remainder ? 1 : 0);
    int thread_col_num = col_end - col_start;
    if (thread_col_num <= 0) return;

    // 局部最大值数组（同基础版）
    float* local_max = (float*)alloca(thread_col_num * sizeof(float));
    for (int i = 0; i < thread_col_num; ++i) {
        local_max[i] = -std::numeric_limits<float>::max();
    }

    // 核心新增：行分块优化，保证块内数据常驻L2/L3缓存
    for (int i_block = 0; i_block < M; i_block += ROW_BLOCK_SIZE) {
        int block_end = std::min(i_block + ROW_BLOCK_SIZE, M);
        // 块内行循环，数据完全命中缓存
        for (int i = i_block; i < block_end; ++i) {
            const float* row_ptr = input + i * N;
            // 最内层连续列访问，顺序读，Cache Line 100%利用
            #pragma GCC unroll 8 // 循环展开，进一步提升指令流水线效率
            for (int j = 0; j < thread_col_num; ++j) {
                float val = row_ptr[col_start + j];
                local_max[j] = std::max(local_max[j], val);
            }
        }
    }

    // 写回结果（同基础版）
    for (int j = 0; j < thread_col_num; ++j) {
        output[col_start + j] = local_max[j];
    }
}
```

#### 🟢 极致性能优化版（SP/校招终面加分项）
在进阶版之上，针对现代CPU架构做极致优化，完全榨干CPU算力和内存带宽：
1.  **SIMD向量化**：使用AVX2/AVX512指令集，一次计算8/16个float的最大值，单指令多数据，提升计算吞吐量
2.  **线程CPU绑定**：将线程绑定到固定CPU核心，避免线程调度导致的缓存失效
3.  **内存对齐**：保证输入矩阵按Cache Line对齐，提升内存访问效率
4.  **预取指令**：主动预取下一行数据，隐藏内存访问延迟

```cpp
#include <immintrin.h> // AVX2头文件

#define ROW_BLOCK_SIZE 512
#define AVX2_FLOAT_NUM 8 // AVX2一次处理8个float

void func_extreme(const float* __restrict__ input, float* __restrict__ output, int M, int N, int n, int tid) {
    assert(input != nullptr && output != nullptr);
    assert(M > 0 && N > 0 && n > 0 && tid >= 0 && tid < n);

    // 线程绑定CPU核心（减少调度开销，缓存常驻）
    cpu_set_t cpuset;
    CPU_ZERO(&cpuset);
    CPU_SET(tid % std::thread::hardware_concurrency(), &cpuset);
    pthread_setaffinity_np(pthread_self(), sizeof(cpu_set_t), &cpuset);

    // 连续列划分（同基础版）
    int cols_per_thread = N / n;
    int remainder = N % n;
    int col_start = tid * cols_per_thread + std::min(tid, remainder);
    int col_end = col_start + cols_per_thread + (tid < remainder ? 1 : 0);
    int thread_col_num = col_end - col_start;
    if (thread_col_num <= 0) return;

    // 32字节对齐的局部max数组，适配AVX2指令
    float* local_max = (float*)aligned_alloc(32, thread_col_num * sizeof(float));
    for (int i = 0; i < thread_col_num; ++i) {
        local_max[i] = -std::numeric_limits<float>::max();
    }

    // 向量化处理的列数
    int vec_col_num = (thread_col_num / AVX2_FLOAT_NUM) * AVX2_FLOAT_NUM;
    int remain_col_num = thread_col_num % AVX2_FLOAT_NUM;

    // 行分块+AVX2向量化
    for (int i_block = 0; i_block < M; i_block += ROW_BLOCK_SIZE) {
        int block_end = std::min(i_block + ROW_BLOCK_SIZE, M);
        for (int i = i_block; i < block_end; ++i) {
            const float* row_ptr = input + i * N + col_start;
            // 主动预取下一行数据，隐藏内存延迟
            __builtin_prefetch(row_ptr + N, 0, 1);

            // AVX2向量化循环：一次处理8个列的最大值
            for (int j = 0; j < vec_col_num; j += AVX2_FLOAT_NUM) {
                __m256 row_vec = _mm256_loadu_ps(row_ptr + j);
                __m256 max_vec = _mm256_loadu_ps(local_max + j);
                max_vec = _mm256_max_ps(max_vec, row_vec);
                _mm256_storeu_ps(local_max + j, max_vec);
            }

            // 处理剩余不足8个的列
            for (int j = vec_col_num; j < thread_col_num; ++j) {
                local_max[j] = std::max(local_max[j], row_ptr[j]);
            }
        }
    }

    // 写回结果
    for (int j = 0; j < thread_col_num; ++j) {
        output[col_start + j] = local_max[j];
    }
    free(local_max);
}
```
